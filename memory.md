## PXE Boot With SSH

Goal: boot pxenode1 into a temporary Alpine netboot image that starts SSH, so hardware can be inspected remotely.

Local PXE server:

- Local interface: `enp0s31f6`
- Local PXE/HTTP IP observed: `192.168.1.236`
- Target PXE/AMT IP observed: `192.168.1.218`
- PXE mode observed: UEFI x86_64, `PXEClient:Arch:00007`
- Target NIC MAC observed in PXE logs: `14:b3:1f:23:0f:7a`

Scripts:

```bash
~/vibe/pxenode1/prepare-pxe-assets
~/vibe/pxenode1/start-pxe-server
~/vibe/pxenode1/stop-pxe-server
~/vibe/pxenode1/ssh-pxe
```

The PXE setup uses:

- `dnsmasq` in proxyDHCP mode, so the normal LAN DHCP server still assigns the IP.
- TFTP for first-stage iPXE: `snponly.efi` for UEFI, `undionly.kpxe` for BIOS fallback.
- HTTP on port `8080` for `boot.ipxe`, Alpine kernel/initramfs, and the apkovl overlay.
- Alpine `latest-stable` x86_64 netboot `virt` kernel/initramfs/modloop.
- apkovl overlay generated by `start-pxe-server`.

The apkovl overlay:

- Adds `openssh`.
- Enables `sshd` in the default OpenRC runlevel.
- Installs local public key from `~/.ssh/id_ed25519.pub` into `/root/.ssh/authorized_keys`.
- Disables password SSH auth.
- Calls back to the local HTTP server:

```text
/booted?host=pxenode1&attempt=1&kernel=<kernel>&ssh=started
```

Successful boot evidence from 2026-06-12:

```text
GET /boot.ipxe
GET /alpine/vmlinuz-virt
GET /alpine/initramfs-virt
GET /pxenode1.apkovl.tar.gz
BOOTED host=pxenode1&attempt=1&kernel=6.18.35-0-virt&ssh=started
```

Boot and SSH procedure:

```bash
~/vibe/pxenode1/start-pxe-server
~/vibe/pxenode1/reset --yes
tail -f ~/vibe/pxenode1/pxe/run/http-requests.log
~/vibe/pxenode1/ssh-pxe
```

To run one command over SSH:

```bash
~/vibe/pxenode1/ssh-pxe 'uname -a; cat /etc/alpine-release; rc-service sshd status'
```

The `ssh-pxe` script uses a temporary known-hosts file at `/tmp/pxenode1-pxe-known-hosts`, because this diskless boot can regenerate host keys.

## Hardware Inspection

Use the PXE SSH image whenever pxenode1 hardware details are needed.

After SSH login, install transient tools if they are not already present:

```bash
apk update
apk add pciutils dmidecode usbutils util-linux ethtool
```

Useful hardware commands:

```bash
lspci -nn
lspci -vvv
dmidecode
lscpu
lsblk -o NAME,MODEL,SERIAL,SIZE,TYPE,FSTYPE,MOUNTPOINTS
ip -brief addr
ethtool -i eth0 2>/dev/null || true
dmesg | grep -Ei 'firmware|microcode|nvme|sata|ahci|ethernet|link'
```

Remember that the Alpine PXE system is temporary/diskless. Changes made after SSH login are lost on the next boot unless added to the apkovl generation in `~/vibe/pxenode1/start-pxe-server`.

## PXE Hello UEFI Test

There is also a minimal UEFI payload for proving that pxenode1 can PXE boot a custom program and print directly to the screen.

Files:

```bash
~/vibe/pxenode1/hello-efi/hello.c
~/vibe/pxenode1/build-hello-efi
~/vibe/pxenode1/start-pxe-hello
```

Build and serve the hello profile:

```bash
~/vibe/pxenode1/stop-pxe-server
~/vibe/pxenode1/start-pxe-hello
~/vibe/pxenode1/reset --yes
```

The `hello` profile writes `boot.ipxe` like this:

```ipxe
#!ipxe
dhcp || goto failed
set base http://192.168.1.236:8080
chain ${base}/hello.efi || goto failed
```

The EFI program calls UEFI `ConOut->ClearScreen()` and `ConOut->OutputString()` to print:

```text
hello from pxenode1 PXE
Tiny UEFI program loaded over PXE/iPXE.
```

Successful fetch evidence from 2026-06-12:

```text
GET /boot.ipxe
GET /hello.efi
```

This profile does not boot Linux and does not start SSH. To return to the SSH-capable Alpine image, restart the normal PXE server:

```bash
~/vibe/pxenode1/stop-pxe-server
~/vibe/pxenode1/start-pxe-server
~/vibe/pxenode1/reset --yes
```
